# Expertise Proposal Writer

Use this skill to improve readability and reviewer utility of proposals generated by `mah expertise propose`.

## Goal

Refine proposal language while keeping evidence, governance posture, and change intent intact.

## Input

- Governance proposal object with:
  - `summary`
  - `rationale`
  - `proposed_changes`
  - `target_snapshot`
  - `evidence_refs`
  - `target_expertise_id`

## Output Rules

- Keep facts unchanged; do not invent evidence or outcomes.
- Keep proposal state unchanged (draft, manual approval required).
- Preserve the technical intent of `proposed_changes`.
- Improve:
  - `summary`: concise and decision-focused
  - `rationale`: explicit evidence-to-change reasoning
  - `proposed_changes`: clear, reviewer-friendly structure
- Use compact, operational writing; avoid marketing language.

## Non-Goals

- Do not modify proposal IDs or governance status.
- Do not alter the actual decision semantics of change fields.
- Do not add unsupported confidence claims.
