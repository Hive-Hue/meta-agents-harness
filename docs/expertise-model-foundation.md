# Expertise Model Foundation (v0.3.0)

## What it is in this release

In `v0.3.0`, the Expertise Model is a first-class foundation concept that standardizes how expertise artifacts are named, referenced, and projected across runtimes.

This release consolidates:

- terminology (`mental model` -> `expertise model`)
- command/tool naming (`update mental model` -> `update expertise model`)
- file/path naming consistency across generated runtime artifacts
- documentation and graph visibility aligned to the new naming

## What it is not in this release

The Expertise Model in `v0.3.0` is not yet an orchestration intelligence engine.

It does not introduce:

- expertise-aware delegation decisions
- confidence/scoring models
- lifecycle governance workflows
- federation/export intelligence
- confidential expertise policy engine
- metrics-driven routing behavior

## Relationship to memory and notes

- Memory keeps context over time.
- Free-form notes capture unstructured observations.
- Expertise Model (current phase) provides a normalized structural reference for expertise assets.

In this phase, expertise is represented consistently and made visible; operational intelligence based on expertise is explicitly deferred.

## Why this scope is intentional

`v0.3.0` is a stabilization/hardening release.  
The goal is conceptual clarity and structural readiness, not expansion into a new feature family.
