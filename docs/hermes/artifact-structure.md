# Hermes Artifact Structure

## Overview

The `.hermes/` directory contains all Hermes-specific runtime artifacts generated by MAH's canonical config projection pipeline.

This directory is created and maintained automatically by `mah init --runtime hermes` and `npm run sync:meta`.

---

## Directory layout

```
.hermes/
├── crew/
│   └── <crew-id>/
│       ├── config.yaml          # Crew-level Hermes configuration
│       ├── multi-team.yaml      # Projected crew topology document
│       ├── agents/
│       │   └── <agent-id>.md    # Agent prompt files
│       ├── expertise/
│       │   └── <agent>-expertise-model.yaml  # Agent expertise files
│       └── sessions/
│           └── <session-id>/    # Session-specific artifacts
└── skills/                      # Hermes-compatible skill definitions
    └── <skill-name>/
        └── SKILL.md
```

---

## Key files

### `config.yaml`

Crew-level Hermes configuration, projected from `meta-agents.yaml` under the `runtimes.hermes` section.

```yaml
# .hermes/crew/dev/config.yaml
runtime: hermes
crew: dev
source_of_truth: meta-agents.yaml
```

### `multi-team.yaml`

The main crew topology document, projected from the canonical config. Contains:

- Crew metadata (mission, sprint mode)
- Orchestrator definition
- Team structure (leads + workers)
- Agent prompts, expertise paths, and skills
- Model assignments and fallbacks
- Domain profiles

This file is deterministic and drift-checkable via `mah diff` or `npm run check:meta-sync`.

### Agent prompt files

Located under `.hermes/crew/<crew>/agents/<agent-id>.md`, these are Hermes-native prompt definitions projected from the canonical config.

Each prompt file includes:
- Agent role and team assignment
- Model and fallback configuration
- Tool and skill references
- Domain access rules
- Sprint context and responsibilities

### Expertise files

Located under `.hermes/crew/<crew>/expertise/`, these YAML files track durable learnings per agent.

---

## Config pattern

Hermes configuration follows the pattern defined in `meta-agents.yaml`:

```yaml
runtimes:
  hermes:
    wrapper: "hermesh"
    direct_cli: "hermes"
    config_root: ".hermes"
    config_pattern: ".hermes/crew/<crew>/config.yaml"
    capabilities:
      persistent_memory: true
      supports_background_operation: true
      supports_multi_backend_execution: true
      gateway_aware: true
```

## Canonical config integration

Hermes artifacts are generated from `meta-agents.yaml` through the sync pipeline:

```bash
# Generate Hermes artifacts from canonical config
npm run sync:meta

# Check for drift without writing
npm run check:meta-sync

# Preview planned changes
mah plan

# Show diff of planned changes
mah diff
```

All generated artifacts are:
- **Deterministic**: same config always produces the same output
- **Diffable**: changes are trackable via git
- **Validatable**: drift is detectable via sync checks

---

## Marker detection

The `.hermes/` directory serves as the Hermes runtime marker. When MAH detects this directory, it recognizes Hermes as a potential runtime:

```
$ mah detect
runtime=hermes
reason=marker
```
