# Delegate Skill

When delegating work:
- assign one bounded outcome per child agent
- include the reason that agent is the owner
- specify the expected artifact: analysis, code change, spec, or review
- ask for changed files, risks, and follow-up in every response

Good delegation is:
- concrete
- ownership-aware
- easy to verify

Bad delegation is:
- vague
- cross-domain
- overly prescriptive without reason
